Front-end Sass Grunt Setup Bootstrap Framework (Windows OS)

Initial Setup

Install Nodejs

Download Node v4.2.2 LTS Windows Installer (.msi) select windows bit version 32-bit/64-bit

Install Git

Download Latest version and install Git.

Add Path Setup

  1. click Start button
  2. right click Computer select Properties
  3. click Advanced system settings
  4. in Advanced Tab click Environment Variables
  5. in System Variables find and click Path then click Edit button
  6. inside Variable value add at the end of Path ;C:\Program Files\nodejs;C:\Users\User Name\AppData\Roaming\npm; click OK button.
  7. Done Good Job!!

Open Command Prompt or Git Bash

type npm install -g bower component to use bootstrap-sass.

type npm install -g grunt-cli for Grunt.js installation

Project Setup

Inside the command prompt or Git Bash, navigate to your project folder directory:

cd C:\Users\Me\MyProject

and type the following commands:

bower init

After this command you are required to enter information / details regarding the project. Feel free to fill it up or not.

bower install bootstrap-sass --save

This commands are used to initialise bower inside your project folder (which in this case is MyProject) also to include bootstrap sass on the project.

npm init

After this command you are required to enter information / details regarding the project. Feel free to fill it up or not.

Next, we create the necessary folders for our development.

  • mkdir js
  • mkdir css
  • mkdir scripts
  • mkdir scss

or Create the following folders inside MyProject.

  • js
  • scss
  • scripts
  • scss

Grunt Setup

Next, we need to install grunt and all the required modules / plugins that we need.

npm install grunt

setup for package.json devDependencies grunt file.

npm install matchdep --save-dev

Use globule to filter npm module dependencies by name.

npm install grunt-contrib-copy --save-dev

Use for copying files and folders

npm install grunt-contrib-uglify --save-dev

Use for minify files with UglifyJS.

npm install grunt-contrib-watch --save-dev

Use for running predefined tasks whenever watched file patterns are added, changed or deleted.

npm install grunt-open --save-dev

Use for opening urls and files from a grunt task.

npm install grunt-express --save-dev

Use for starting (and supervise) an Express.js web server using grunt.js.

npm install grunt-parallel --save-dev

Use for running tasks or commands in child processes.

npm install grunt-rev --save-dev

Use for Static file asset revisioning through content hashing.

npm install grunt-sass --save-dev

Use for compiling Sass to CSS using node-sass.

Create Grunt file (Gruntfile.js) with this code:

// Start

module.exports = function(grunt){
    // Load Grunt tasks declared in the package.json file
    require('matchdep').filterDev('grunt-*').forEach(grunt.loadNpmTasks);

    grunt.initConfig({
        rev : {
            options : {
                encoding : 'utf8',
                algorithm : 'md5',
                length : 5
            },
            assets : {
                files : [{
                    src : [
                        'js/*.js',
                        'css/*.css'
                    ]
                }]
            }
        },
        // copy files
        copy: {
            main: {
                files: [
                    // Vendor scripts.
                    {
                        expand: true,
                        cwd: 'bower_components/bootstrap-sass/assets/javascripts/',
                        src: ['**/*.js'],
                        dest: 'scripts/bootstrap-sass/'
                    },
                    {
                        expand: true,
                        cwd: 'bower_components/jquery/dist/',
                        src: ['**/*.js', '**/*.map'],
                        dest: 'scripts/jquery/'
                    },

                    // Fonts.
                    {
                        expand: true,
                        filter: 'isFile',
                        flatten: true,
                        cwd: 'bower_components/',
                        src: ['bootstrap-sass/assets/fonts/**'],
                        dest: 'fonts/'
                    },

                    // Stylesheets
                    {
                        expand: true,
                        cwd: 'bower_components/bootstrap-sass/assets/stylesheets/',
                        src: ['**/*.scss'],
                        dest: 'scss/'
                    }
                ]
            },
        },

        // SASS compilation
        // Compile SASS files into minified CSS.
        sass: {
            options: {
                includePaths: ['bower_components/bootstrap-sass/assets/stylesheets']
            },
            dist: {
                options: {
                    outputStyle: 'compressed'
                },
                files: {
                    'css/app.css': 'scss/app.scss'
                }
            }
        },

        // uglify
        uglify : {
            options : {
                mangle : false
            },
            my_target : {
                files : {'js/app.js' : ['scripts/*.js'] }
            }
        },

        // The webserver
        express : {
            all : {
                options :{
// Set your file directory 
                    bases : ['C:\\Users\\Me\\MyProject'],
                    port : 9000,
                    hostname : "0.0.0.0",
                    livereload : true
                }
            }
        },

        watch : {
            all : {
                files : ['Gruntfile.js','scripts/*.js', 'index.html'],
                tasks : ['uglify'],
                options : {
                    livereload : true
                }
            },
            css : {
                files : ['scss/*.scss','scss/**/*.scss','scss/**/**/*.scss'],
                tasks : ['sass']
            }
        },

        open : {
            all : {
                path : 'http://localhost:9000/index.html'
            }
        }
    });

grunt.loadNpmTasks('grunt-sass');
grunt.loadNpmTasks('grunt-express');
grunt.loadNpmTasks('grunt-contrib-watch');
grunt.loadNpmTasks('grunt-contrib-copy');
grunt.loadNpmTasks('grunt-contrib-uglify');
grunt.loadNpmTasks('grunt-rev');

    // Establish tasks we can run from the terminal.
grunt.registerTask('build', ['sass', 'copy', 'uglify', 'express', 'open']);
    grunt.registerTask('default', ['build', 'watch']);
};

// End

Uncomment scss file if will be use and create scss files (ex: _mixins.scss) or add new scss file for other function.

Inside scss folder create folder structure for scss component:

  • base
  • layout
  • modules

In base folder, your scss are for text hierarchy, color palette/themes, mixins functions and custom style button ready to use in website/web-app.

In layout folder, your scss are for layout/theme/page structure.

In modules folder, your scss are for custom modules and special functions inside your layout, examples are (breadcrumbs, modal, panel, table, thumbnail, forms and etc..).

Be organized in scss file folder structure.
